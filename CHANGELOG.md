# Changelog

このプロジェクトの重要な変更はすべてこのファイルに記録します。

書式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に基づきます。

## [Unreleased]

### Added

- ファイルの肥大化とリファクタリングによる退行を防ぐため、1 ファイルを 1000 行以下に保ち、既存の超過ファイルは動的要素テストと機能テストを追加してから責務分離し、分離後にも再検証する共通ルールを追加。
- 共通ルールを集約する `AGENTS.md` を追加。
- リポジトリの目的と使い方を説明する `README.md` を追加。
- 共通ルールの更新手順をまとめた `how-to-update.md` を追加。
- 各リポジトリに `COMMON-AGENTS.md` シンボリックリンクを作成する `scripts/New-CommonAgentsLink.ps1` を追加。
- `COMMON-AGENTS.md` を Git 管理外にする `.gitignore` を追加。
- ケバブケース命名と `any` 回避を共通の実装方針として追加。
- データストレージのミグレーション、バージョンチェック、破損データリカバリ方針を追加。
- 新しいデータモデル追加時は旧データを原則として強制ミグレーション対象にする方針を追加。
- 不要コード、デッドコード、使われなくなった互換処理を積極的に削除するソースコード保守方針を追加。
- Chrome CDP と Firefox RDP / Marionette を使うブラウザ検証方針を追加。
- ブラウザ検証手段をユーザー向け説明へ不要に露出しない方針を追加。
- 作業前確認、依存関係、生成物、秘密情報、エラー対応、ドキュメント更新、テスト設計の共通ルールを追加。
- ユーザー向け UI で日本語と総話者数上位10言語を優先する多言語対応方針を追加。
- CSS セレクター、DOM 探索、E2E テストのロケーターや判定条件を表示言語に依存させない方針を追加。
- React SPA や CSS Modules / CSS-in-JS / UI ライブラリが生成するハッシュ付きクラス名、自動生成 ID、ランダム文字列を含むセレクターを安定した依存先にしない方針を追加。
- 複数の解像度、アスペクト比、縦長環境、モバイル、4K で破綻しない表示品質とレスポンシブ対応方針を追加。
- CSS の `!important` 乱用を避け、既存の乱用箇所を整理する方針を追加。
- サブエージェントの利用条件、委譲範囲、監査ロール、統合責任に関する共通ルールを追加。
- サブエージェント用の一時作業フォルダ `subagents/` を簡易メモリー兼作業分離領域として使う方針を追加。
- E2E テストとユニットテストで動的 UI 要素と状態遷移を網羅的に確認し、自動化困難箇所も同等動作のモック環境で検証する方針を追加。
- Codex App と Codex VS Code IDE の会話履歴分析に基づき、推測実装、検証不足、Git/PR 状態確認漏れ、ユーザー意図の取り違えを防ぐ作業規律を追加。
- 変更履歴から変更の目的とユーザー影響を読み取れるように、`CHANGELOG.md` を意図ベースで記載し、コミット履歴から作成する場合も実際のコード差分から意図を推測する方針を追加。
