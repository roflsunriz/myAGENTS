# AGENTS.md

このリポジトリは、`C:\Users\UserName\Documents` と `C:\filter-matome` 配下の各リポジトリで共通して守る、コーディングエージェント向けの中核ルールを管理する。

各プロジェクト固有の目的、設計、起動方法、例外事項は、そのプロジェクトのローカル `AGENTS.md` に記載する。このファイルでは、全リポジトリで常に優先する作業規律だけを扱う。

## 応答と言語

- エージェントは常に日本語で応答する。
- 技術用語、コマンド名、API 名、エラーメッセージは必要に応じて原文のまま使ってよい。
- 不明点があっても、ローカルの README、`AGENTS.md`、更新手順、設定ファイル、CI 定義を先に確認し、推測だけで作業を進めない。

## コミット

- コミットメッセージは日本語 Conventional Commits 形式にする。
- 形式は `type(scope): 要約` を基本とする。scope が不要な場合は `type: 要約` とする。
- 例:
  - `feat: 更新手順の確認フローを追加`
  - `fix(api): 型エラー時の戻り値を修正`
  - `docs: CHANGELOG の運用方針を追記`
- 変更理由、検証内容、破壊的変更がある場合は本文に日本語で記載する。

## 設計と実装方針

- モノリシックな実装を避け、複数ファイルに責務を分離する。
- 1 ファイルに UI、状態管理、API 通信、データ変換、バリデーション、永続化を過度に詰め込まない。
- 既存のディレクトリ構成、命名、設計思想を優先し、不要な大規模リファクタリングを避ける。
- 成熟した npm / GitHub パッケージが存在する場合は、自前実装より利用を優先する。
- ただし、採用前に以下を調査する。
  - 直近のリリース頻度
  - 直近のコミット頻度
  - issue / PR の放置状況
  - ダウンロード数、スター数、利用実績
  - ライセンス
  - TypeScript 対応や型定義の有無
- 最近のリリースが 1 年以上止まっている、直近のコミットがほとんどない、重大な issue が放置されているなど、保守性に納得できない場合は採用しない。

## 更新手順

- 更新手順は原則として `how-to-update.md` にまとめる。
- プロジェクトで `docs/UPDATE.md` を使っている場合は、それを正式な更新手順として扱ってよい。
- どちらも存在しない場合は、作業中に判明した更新手順を `how-to-update.md` または `docs/UPDATE.md` として作成する。
- 更新手順には、必要な前提、実行コマンド、検証方法、ロールバックまたは復旧方針を含める。

## 変更履歴

- 変更点は `CHANGELOG.md` に記載する。
- 書式は Keep a Changelog に従う。
- 日付は `YYYY-MM-DD` 形式で記載する。
- 未リリースの変更は `## [Unreleased]` に集約する。
- 種別は必要に応じて `Added`、`Changed`、`Deprecated`、`Removed`、`Fixed`、`Security` を使う。

## 検証

TypeScript / Bun プロジェクトでは、原則として以下をローカルで実行する。

```powershell
bun run lint
bun run format
bun run type-check
bun run build
bun run test
```

CI チェックや脆弱性チェックが定義されている場合は、それもローカルで可能な範囲で実行する。

TypeScript ではないプロジェクトでは、言語と構成に応じて次の品質確認を行う。

- 型チェック、静的解析、lint
- フォーマット
- 単体テスト、統合テスト
- ビルドまたはパッケージング
- 構文チェック
- セキュリティスキャンまたは依存関係監査

実行できない検証がある場合は、理由と未検証リスクを明記する。

## ローカル AGENTS.md との関係

- このファイルのルールは全プロジェクト共通の上位方針とする。
- 各プロジェクトのローカル `AGENTS.md` は、プロジェクト固有の情報を補足する。
- 各プロジェクトでは `COMMON-AGENTS.md` をこのファイルへのシンボリックリンクとして配置し、ローカル `AGENTS.md` から必ず参照する。
- `COMMON-AGENTS.md` は各プロジェクトの `.gitignore` に追加し、Git 管理外にする。
- ローカル `AGENTS.md` にこのファイルと矛盾する内容がある場合は、セキュリティ、品質、応答言語、変更履歴、検証に関するこのファイルの方針を優先する。
- 例外が必要な場合は、ローカル `AGENTS.md` に理由、範囲、代替検証を明記する。
